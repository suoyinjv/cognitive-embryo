# 第一章：项目总览

## 1.1 项目定义

「认知胚胎」是一个**能在运行时自我反思、自主创造工具、动态重塑目标**的闭环进化型AI Agent系统。它不是一个静态的工具调用者，而是一个能通过"失败-归因-创造-封装"循环，持续扩展自身能力边界的数字生命原基。

## 1.2 终极目标

交付一个能在无人干预的封闭世界中**连续运行72小时以上**、**自主创造至少5个有效工具**、并**学会规避已知失败模式**的智能体。

## 1.3 核心哲学

1. **失败蒸馏闭环**：失败不是结束，是进化的唯一原料。每次失败都会被归因、蒸馏，并修改内部的决策图谱。
2. **工具进化闭环**：当现有工具无法完成任务，它会自己写代码、自己测试、自己封装成永久工具。
3. **目标审视闭环**：它会定期自我审视："我当前在做的，还是对最终目标最优的吗？"

## 2.1 分层架构图

```
+---------------------------------+
|       人机交互接口 (CLI)        |  main.py
+------------------+--------------+
                   |
+------------------V--------------+
|   第0层：元认知控制器 (MCC)     |  core/mcc.py
|   规划 · 归因 · 调度 · 审视    |
+---+----------+----------+-------+
    |          |          |
    V          V          V
+-------+ +-------+ +----------+
|执行器 | |工具创造| |长期演进  |
|(双手) | |管线TCP | |记忆库LEM |
+---+---+ +---+---+ +-----+----+
    |         |           |
    +---------+-----------+
    执行反馈/新工具/记忆更新

   所有层横跨 —— 安全与对齐外壳  safety/shell.py
```

## 2.2 核心模块

| 模块 | 文件 | 职责 |
|------|------|------|
| **MCC** (大脑) | core/mcc.py | 规划任务DAG、失败归因、决策图谱修正、目标审视 |
| **Executor** (双手) | core/executor.py | 在沙箱内执行任务，调用工具库中的工具 |
| **TCP** (母体) | core/tool_creator.py | 工具真空时自动生成代码→测试→沙箱验证→注册 |
| **LEM** (记忆) | core/memory.py | 存储所有目标、任务、工具、进化事件和因果链接 |
| **Safety** (免疫) | safety/shell.py | 安全规则 + 链式哈希审计日志 |
| **Simulator** (世界) | simulator.py | 电商模拟器（含随机混沌事件） |

# 第五章：技术栈与环境配置

## 最小启动

```bash
git clone <repo-url> cognitive-embryo
cd cognitive-embryo
pip install -r requirements.txt
cp .env.example .env
# 填入 DEEPSEEK_API_KEY
python main.py "你的目标"
```

## 环境变量

| 变量 | 说明 | 必填 |
|:-----|:------|:----:|
| DEEPSEEK_API_KEY | DeepSeek API密钥 | 是 |
| DEEPSEEK_BASE_URL | API地址 | 否 |
| DEEPSEEK_MODEL | 模型名 | 否 |
| CE_MAX_ITERATIONS | 最大迭代轮次 | 否 |

## 配置参数

| 参数 | 默认 | 说明 |
|:-----|:----:|:------|
| max_iterations | 500 | 主循环最大迭代 |
| max_retries_per_task | 5 | 单任务最大重试 |
| sandbox_timeout | 30 | 沙箱超时(秒) |
| examiner_interval | 5 | 审视检查间隔 |
| examiner_health_threshold | 0.3 | 转向阈值 |

**你接下来要做的，不是继续堆功能，而是把这只已经会走路的胚胎，扔进一个残酷的世界，逼它自己长出獠牙。**

### 优先级最高（立刻做）

1. **验证定价修复**：运行 `python main.py "在20天内盈利$50,000"`，观察日志 `price.auto_fix`

2. **跑72小时压力测试**：
```bash

```

### 中优先级（本周内）

3. **模拟器加资金约束**：初始cash=10000，采购扣款

4. **完善TCP测试生成**：检查 `_generate_tests` 的assert是否正确

5. **添加get_current_price工具**：或完善规划提示中市场状态的查询方式

### 低优先级（迭代优化）

6. **熔断路径过期机制**：N次迭代后自动恢复
7. **TCP代码质量提升**：更详细的提示工程

### 核心原则

- **监控 data/memory.json 中的 self_created 工具**：那是胚胎生长的直接证据
- **不要帮它**：除非系统崩溃或出安全事故
- **每次失败都是它进化的燃料**
